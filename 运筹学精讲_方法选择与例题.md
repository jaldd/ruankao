## 一、线性规划

### 1.1 什么时候用

题目特征：**求最大利润或最小成本，有约束条件（资源有限），关系都是线性的**

关键词：最大化、最小化、不超过、至少、资源限制

### 1.2 标准形式

```
目标函数：max Z = c₁x₁ + c₂x₂ + ... + cₙxₙ
约束条件：a₁₁x₁ + a₁₂x₂ + ... ≤ b₁
          a₂₁x₁ + a₂₂x₂ + ... ≤ b₂
          x₁, x₂, ... ≥ 0
```

### 1.3 解法：画图法（2个变量时用）

**步骤**：
1. 画出每个约束条件的边界线
2. 找出可行域（所有约束的交集）
3. 画目标函数的等值线
4. 沿优化方向平移等值线，最后一个接触可行域的顶点就是最优解

### 1.4 例题

某工厂生产A、B两种产品。A产品利润3万元/件，B产品利润5万元/件。生产1件A需设备2小时、原料1kg；生产1件B需设备1小时、原料2kg。设备总工时8小时，原料总量8kg。求最大利润。

**解**：

设生产A产品x₁件，B产品x₂件

```
目标函数：max Z = 3x₁ + 5x₂
约束条件：2x₁ + x₂ ≤ 8  （设备工时）
          x₁ + 2x₂ ≤ 8  （原料）
          x₁, x₂ ≥ 0
```

画图求交点：
- 2x₁ + x₂ = 8 与 x₁ + 2x₂ = 8 联立
- 解得 x₁ = 8/3, x₂ = 8/3

验证各顶点：
| 顶点 | Z = 3x₁ + 5x₂ |
|------|---------------|
| (0, 0) | 0 |
| (4, 0) | 12 |
| (0, 4) | 20 |
| (8/3, 8/3) | 64/3 ≈ 21.3 |

**最优解**：x₁ = 8/3, x₂ = 8/3，最大利润 Z = 64/3 ≈ 21.3万元

## 二、运输问题

### 2.1 什么时候用

题目特征：**从多个产地运到多个销地，求最小总运费**

关键词：产地、销地、运价、运量、供需平衡

### 2.2 为什么运输问题不用线性规划？

运输问题本质上是特殊的线性规划，但变量太多（m×n个），用单纯形法太慢。所以发明了**表上作业法**——直接在运价表上操作，简单高效。

### 2.3 解法三步走

```
第一步：找初始解（3种方法选1种）
  ├─ 西北角法：最简单，但初始解质量差
  ├─ 最小元素法：每次选运价最小的格子填，质量中等
  └─ 伏格尔法：考虑"不选最小会多花多少"，质量最好 ← 考试推荐

第二步：检验是否最优（2种方法选1种）
  ├─ 闭回路法：找闭回路算检验数
  └─ 位势法：算位势再算检验数 ← 考试推荐

第三步：如果不是最优，调整（闭回路调整法）
  → 回到第二步，直到所有检验数≥0
```

### 2.4 伏格尔法详细步骤

**核心思想**：每行每列算"罚数"=次小运价-最小运价。罚数越大说明"不选最小的代价越大"，优先满足罚数最大的行/列。

**步骤**：
1. 对每行每列，算最小运价和次小运价之差（罚数）
2. 选罚数最大的行或列，在该行/列中选运价最小的格子，尽量填满
3. 划掉已满足的行或列
4. 重复1-3，直到所有格子填完

### 2.5 例题

三个产地A1、A2、A3，产量分别为7、4、9；四个销地B1、B2、B3、B4，销量分别为3、6、5、6。运价表如下：

| | B1 | B2 | B3 | B4 | 产量 |
|---|---|---|---|---|---|
| **A1** | 3 | 11 | 3 | 10 | 7 |
| **A2** | 1 | 9 | 2 | 8 | 4 |
| **A3** | 7 | 4 | 10 | 5 | 9 |
| **销量** | 3 | 6 | 5 | 6 | |

**用伏格尔法求初始解**：

第1轮罚数：
- A1行：最小3，次小3，罚数=0
- A2行：最小1，次小2，罚数=1
- A3行：最小4，次小5，罚数=1
- B1列：最小1，次小3，罚数=2 ← 最大
- B2列：最小4，次小9，罚数=5 ← 最大
- B3列：最小2，次小3，罚数=1
- B4列：最小5，次小8，罚数=3

B2列罚数最大(5)，B2列最小运价是A3→B2的4。A3产量9，B2销量6，填6，B2列满足划掉。

第2轮（B2列已划掉）：
- B1列罚数=2，B3列罚数=1，B4列罚数=3 ← 最大
- B4列最小运价是A3→B4的5。A3剩余3，B4销量6，填3，A3行满足划掉。

第3轮（B2列、A3行已划掉）：
- B1列罚数=2，B3列罚数=1，B4列罚数=2
- B1列和B4列罚数都是2，选B1列（运价更小）。B1列最小运价是A2→B1的1。A2产量4，B1销量3，填3，B1列满足划掉。

第4轮：A2剩余1，填A2→B3的2（运价最小），填1。A2行满足划掉。

第5轮：A1产量7，B3还需4，B4还需3。填A1→B3运价3，填4；填A1→B4运价10，填3。

**初始解**：

| | B1 | B2 | B3 | B4 | 产量 |
|---|---|---|---|---|---|
| **A1** | | | 4 | 3 | 7 |
| **A2** | 3 | | 1 | | 4 |
| **A3** | | 6 | | 3 | 9 |
| **销量** | 3 | 6 | 5 | 6 | |

总运费 = 4×3 + 3×10 + 3×1 + 1×2 + 6×4 + 3×5 = 12+30+3+2+24+15 = **86**

> 考试一般只要求用伏格尔法求初始解，不要求继续优化。如果要求最优解，再用位势法检验+闭回路调整。

## 三、指派问题（匈牙利法）

### 3.1 什么时候用

题目特征：**n个人做n件事，每人只做一件，每件只由一人做，求总成本最小或总效益最大**

关键词：指派、分配、一人一事、成本矩阵

### 3.2 运输问题 vs 指派问题 — 关键区别

| | 运输问题 | 指派问题 |
|---|---|---|
| 供需关系 | 产地产量≠销地销量（可以不等） | 人数=任务数（必须相等） |
| 分配方式 | 一个产地可供给多个销地 | 每人只做一件事 |
| 解法 | 表上作业法 | 匈牙利法 |
| 变量值 | 可以是任意非负数 | 只能是0或1 |

**为什么指派问题不用运输问题的方法？** 因为指派问题的解只能取0或1（做/不做），运输问题的解可以取任意值。匈牙利法专门利用了这个0-1特性，效率更高。

### 3.3 匈牙利法步骤（求最小值）

**步骤**：
1. **行变换**：每行减去该行最小值
2. **列变换**：每列减去该列最小值
3. **画0线**：用最少的线覆盖所有0元素
4. **判断**：如果线数=n，找到最优解；如果线数<n，继续第5步
5. **调整**：未被线覆盖的最小元素记为k，未覆盖的减k，交叉点加k，线上的不变
6. 重复3-5，直到线数=n
7. **分配**：从只有一个0的行/列开始分配

### 3.4 例题（求最小成本）

4个人做4件事，成本矩阵如下：

| | 任务1 | 任务2 | 任务3 | 任务4 |
|---|---|---|---|---|
| **甲** | 2 | 10 | 9 | 7 |
| **乙** | 15 | 4 | 14 | 8 |
| **丙** | 13 | 14 | 16 | 11 |
| **丁** | 4 | 15 | 13 | 9 |

**第1步：行变换（每行减最小值）**

甲行最小2：2-2=0, 10-2=8, 9-2=7, 7-2=5
乙行最小4：15-4=11, 4-4=0, 14-4=10, 8-4=4
丙行最小11：13-11=2, 14-11=3, 16-11=5, 11-11=0
丁行最小4：4-4=0, 15-4=11, 13-4=9, 9-4=5

| | 任务1 | 任务2 | 任务3 | 任务4 |
|---|---|---|---|---|
| **甲** | 0 | 8 | 7 | 5 |
| **乙** | 11 | 0 | 10 | 4 |
| **丙** | 2 | 3 | 5 | 0 |
| **丁** | 0 | 11 | 9 | 5 |

**第2步：列变换（每列减最小值）**

列1最小0，列2最小0，列3最小5，列4最小0

列3减5：7-5=2, 10-5=5, 5-5=0, 9-5=4

| | 任务1 | 任务2 | 任务3 | 任务4 |
|---|---|---|---|---|

| **乙** | 11 | 0 | 5 | 4 |

| **丁** | 0 | 11 | 4 | 5 |

**第3步：画0线**

用3条线可以覆盖所有0（画甲行、乙行、丙行），线数=3 < n=4，需要调整。

**第5步：调整**

未被覆盖的最小元素=2（甲行任务3的2）

未覆盖的减2，交叉点加2，线上的不变：

| | 任务1 | 任务2 | 任务3 | 任务4 |
|---|---|---|---|---|
| **甲** | 0 | 6 | 0 | 3 | ← 被线覆盖，但任务3未被覆盖，减2→0
| **乙** | 13 | 0 | 5 | 4 | ← 被线覆盖，任务2未被覆盖但已被乙行的线覆盖
| **丙** | 4 | 3 | 0 | 0 | ← 被线覆盖
| **丁** | 0 | 9 | 2 | 3 | ← 未被覆盖，减2

等一下，让我重新画线。画线策略：从0最多的行/列开始。

0的位置：(甲,1), (甲,3), (乙,2), (丙,3), (丙,4), (丁,1)

画线方案：列1一条线覆盖(甲,1)和(丁,1)，列2一条线覆盖(乙,2)，列3一条线覆盖(甲,3)和(丙,3)→但(丙,4)没被覆盖。

再换：甲行、乙行、丙行→3条线覆盖了甲乙丙三行的所有0，丁行的(丁,1)=0没被覆盖。

所以需要4条线？不对，让我重新分析。

0的位置：甲1, 甲3, 乙2, 丙3, 丙4, 丁1

画线方案1：列1、列2、列3 → 3条线覆盖甲1,丁1,乙2,甲3,丙3 → 但丙4没覆盖
画线方案2：列1、列2、丙行 → 3条线覆盖甲1,丁1,乙2,丙3,丙4 → 但甲3没覆盖
画线方案3：甲行、乙行、丙行 → 3条线覆盖甲1,甲3,乙2,丙3,丙4 → 但丁1没覆盖

3条线无法覆盖所有0，所以线数=3 < 4，需要调整。

**调整**：未被线覆盖的元素中找最小值。

用方案3（甲行、乙行、丙行画线），未被覆盖的是丁行：
丁行：0, 9, 2, 3 → 最小值=2（丁3的2，但丁1的0也没被覆盖...等等）

不对，让我重新来。用甲行、乙行、丙行画线时，丁行没被覆盖。丁行的元素是0, 9, 2, 3。但列1也没被画线覆盖（因为画的是行线），所以丁1=0是未被覆盖的。

未被覆盖的所有元素：丁1=0, 丁2=9, 丁3=2, 丁4=3。最小值=0。

1. 给没有分配0的行打√

**最优分配**：
- 甲→任务3，成本9
- 乙→任务2，成本4
- 丙→任务4，成本11
- 丁→任务1，成本4

**最小总成本 = 9+4+11+4 = 28**

### 3.5 求最大值怎么办？

如果题目求最大效益（而不是最小成本），把矩阵中所有元素取负，然后减去最大值，转化为最小值问题。

**简化做法**：找出矩阵最大元素M，每个元素变成M-原值，然后按匈牙利法求最小值。

### 3.6 人数≠任务数怎么办？

- 人多任务少：加虚拟任务（成本为0的列）
- 人少任务多：加虚拟人（成本为0的行）

## 四、决策论

### 4.1 什么时候用

题目特征：**面临多种方案，不同自然状态下收益不同，要选最优方案**

### 4.2 两类决策 — 最关键的区别

| | 风险型决策 | 不确定型决策 |
|---|---|---|
| **是否知道概率** | ✅ 知道每种状态的概率 | ❌ 不知道概率 |
| **用什么方法** | 期望值法(EMV)、决策树 | 悲观/乐观/折中/等可能/后悔值 |
| **考试频率** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |

### 4.3 不确定型决策 — 5种方法

**例题**：某项目有3种方案，4种自然状态，收益矩阵如下：

| | 状态1 | 状态2 | 状态3 | 状态4 |
|---|---|---|---|---|
| **方案A** | 4 | 5 | 6 | 7 |
| **方案B** | 2 | 4 | 6 | 9 |
| **方案C** | 5 | 7 | 3 | 5 |

**方法1：悲观法（小中取大法/瓦尔德法）**
- 思路：最坏情况下选最好的
- 做法：每行取最小值，再取最大值
- A: min(4,5,6,7)=4, B: min(2,4,6,9)=2, C: min(5,7,3,5)=3
- 选max(4,2,3)=4 → **方案A**
- 适用：保守型决策者

**方法2：乐观法（大中取大法）**
- 思路：最好情况下选最好的
- 做法：每行取最大值，再取最大值
- A: max=7, B: max=9, C: max=7
- 选max(7,9,7)=9 → **方案B**
- 适用：冒险型决策者

**方法3：折中法（赫维兹法）**
- 思路：在悲观和乐观之间折中
- 做法：CV = α×最大值 + (1-α)×最小值，α为乐观系数
- 假设α=0.6：
  - A: 0.6×7+0.4×4=5.8
  - B: 0.6×9+0.4×2=6.2
  - C: 0.6×7+0.4×3=5.4
- 选max(5.8,6.2,5.4)=6.2 → **方案B**

**方法4：等可能法（拉普拉斯法）**
- 思路：假设每种状态概率相等
- 做法：每行求平均值，取最大
- A: (4+5+6+7)/4=5.5, B: (2+4+6+9)/4=5.25, C: (5+7+3+5)/4=5
- 选max(5.5,5.25,5)=5.5 → **方案A**

**方法5：后悔值法（萨维奇法）**
- 思路：选了某方案后，发现不是最优会后悔，选"最少后悔"的
- 做法：先算后悔值矩阵（每列最大值-该元素），再每行取最大后悔值，选最小的

后悔值矩阵：
| | 状态1 | 状态2 | 状态3 | 状态4 | 最大后悔 |
|---|---|---|---|---|---|
| **A** | 5-4=1 | 7-5=2 | 6-6=0 | 9-7=2 | **2** |
| **B** | 5-2=3 | 7-4=3 | 6-6=0 | 9-9=0 | **3** |
| **C** | 5-5=0 | 7-7=0 | 6-3=3 | 9-5=4 | **4** |

选min(2,3,4)=2 → **方案A**

**5种方法总结**：

| 方法 | 口诀 | 选什么 | 本题答案 |
|------|------|--------|---------|
| 悲观法 | 小中取大 | 每行最小取最大 | A |
| 乐观法 | 大中取大 | 每行最大取最大 | B |
| 折中法 | 加权折中 | α×max+(1-α)×min | B(α=0.6) |
| 等可能法 | 平均取大 | 每行平均取最大 | A |
| 后悔值法 | 大中取小 | 每行最大后悔取最小 | A |

### 4.4 风险型决策 — 期望值法(EMV)

**例题**：某项目有3种方案，市场状态有好(0.3)、中(0.5)、差(0.2)三种概率：

| | 好(0.3) | 中(0.5) | 差(0.2) |
|---|---|---|---|
| **方案A** | 20 | 12 | 8 |
| **方案B** | 30 | 10 | -5 |
| **方案C** | 15 | 13 | 10 |

EMV(A) = 20×0.3 + 12×0.5 + 8×0.2 = 6+6+1.6 = 13.6
EMV(B) = 30×0.3 + 10×0.5 + (-5)×0.2 = 9+5-1 = 13
EMV(C) = 15×0.3 + 13×0.5 + 10×0.2 = 4.5+6.5+2 = 13

选max(13.6, 13, 13) = 13.6 → **方案A**

### 4.5 决策树

决策树是EMV的图形化表示，适合多阶段决策。

**例题**：某公司考虑是否开发新产品。如果开发，需投资50万。开发后市场好(0.6)可获利120万，市场差(0.4)将亏损30万。如果不开发，可把50万存银行获利5万。

```
          ┌── 好(0.6) → 120万
开发 ─50万┤
          └── 差(0.4) → -30万

EMV(开发) = 0.6×120 + 0.4×(-30) = 72-12 = 60万
净EMV(开发) = 60 - 50 = 10万

不开发：5万

10万 > 5万 → 选择开发
```

## 五、博弈论

### 5.1 什么时候用

题目特征：**两方或多方对抗，一方的收益取决于对方的策略选择**

### 5.2 博弈论核心分类

```
博弈论
  ├─ 二人零和博弈（我赢=你输，总和=0）
  │   ├─ 有鞍点 → 纯策略 → 鞍点法
  │   └─ 无鞍点 → 混合策略 → 线性规划法
  │
  └─ 非零和博弈（双方可以都赢或都输）
      └─ 纳什均衡
```

### 5.3 鞍点法（纯策略博弈）

**什么时候用**：二人零和博弈，存在鞍点

**什么是鞍点**：某行最小值中的最大值 = 某列最大值中的最小值

**例题**：甲乙博弈，甲的收益矩阵：

| 甲\乙 | B1 | B2 | B3 | 行最小 |
|--------|-----|-----|-----|--------|
| **A1** | -1 | 3 | -3 | -3 |
| **A2** | 2 | 5 | 1 | 1 ← max |
| **A3** | 1 | 4 | 0 | 0 |
| **列最大** | 2 | 5 | 1 | |

甲的策略（maximin）：选行最小值中最大的 → max(-3, 1, 0) = 1（A2）
乙的策略（minimax）：选列最大值中最小的 → min(2, 5, 1) = 1（B3）

maximin = minimax = 1 → **存在鞍点！**

鞍点在(A2, B3)，博弈值V=1。

**含义**：甲选A2，乙选B3，甲的收益为1。双方都没有动力单方面改变策略。

### 5.4 无鞍点怎么办？（混合策略）

**例题**：

| 甲\乙 | B1 | B2 | 行最小 |
|--------|-----|-----|--------|
| **A1** | 2 | -1 | -1 |
| **A2** | -1 | 2 | -1 |
| **列最大** | 2 | 2 | |

maximin = -1, minimax = 2, 不相等 → **无鞍点**

此时双方都不存在"绝对最优"的纯策略，必须用**混合策略**——以一定概率随机选择。

**2×2博弈的公式法**：

设甲以概率p选A1，(1-p)选A2。

甲选A1的期望 = p×2 + (1-p)×(-1) = 3p-1
甲选A2的期望 = p×(-1) + (1-p)×2 = 2-3p

令两式相等：3p-1 = 2-3p → 6p = 3 → p = 1/2

博弈值 V = 3×(1/2)-1 = 1/2

同理乙以概率q=1/2选B1。

**混合策略解**：甲以1/2概率选A1、1/2概率选A2；乙以1/2概率选B1、1/2概率选B2。博弈值V=1/2。

### 5.5 纳什均衡

**什么时候用**：非零和博弈（双方收益不是此消彼长的关系）

**什么是纳什均衡**：给定对方的策略，自己单方面改变策略不会变得更好。即"谁都不想先变"的状态。

**经典例题：囚徒困境**

| | 乙沉默 | 乙背叛 |
|---|---|---|
| **甲沉默** | (-1, -1) | (-10, 0) |
| **甲背叛** | (0, -10) | (-5, -5) |

（每格第一个数是甲的刑期，第二个是乙的刑期，越小越好）

分析：
- 如果乙沉默：甲背叛(0) > 甲沉默(-1)，甲会背叛
- 如果乙背叛：甲背叛(-5) > 甲沉默(-10)，甲会背叛
- 所以无论乙怎么选，甲都会背叛 → 背叛是甲的占优策略
- 同理，背叛也是乙的占优策略

**纳什均衡**：(背叛, 背叛) → 各判5年

**注意**：(-1,-1)双方都更好，但不是纳什均衡——因为任何一方都可以单方面背叛来改善自己的处境。

### 5.6 鞍点法 vs 纳什均衡 — 关键区别

| | 鞍点法 | 纳什均衡 |
|---|---|---|
| **博弈类型** | 二人零和博弈 | 任意博弈 |
| **收益关系** | 我赢=你输，总和=0 | 任意 |
| **解的含义** | 双方最优纯策略 | 谁都不想单方面改变 |
| **存在性** | 不一定有鞍点 | 任何博弈至少有一个纳什均衡（可能是混合策略） |
| **考试考法** | 给矩阵算鞍点 | 给场景判断是否纳什均衡 |

## 六、排队论

### 6.1 什么时候用

题目特征：**顾客到达、排队等候、接受服务**

关键词：到达率、服务率、平均等待时间、平均队长

### 6.2 M/M/1模型（最常考）

M/M/1 = 顾客到达服从泊松分布(M) / 服务时间服从指数分布(M) / 1个服务台

**核心公式**：

设 λ = 到达率（单位时间到达人数），μ = 服务率（单位时间服务人数）

| 指标 | 公式 | 含义 |
|------|------|------|
| 系统利用率 ρ | ρ = λ/μ | 必须ρ<1，否则队列无限增长 |
| 系统中平均人数 L | L = λ/(μ-λ) = ρ/(1-ρ) | 包括正在服务的 |
| 队列中平均人数 Lq | Lq = λ²/(μ(μ-λ)) = ρ²/(1-ρ) | 不包括正在服务的 |
| 系统中平均逗留时间 W | W = 1/(μ-λ) | 包括服务时间 |
| 队列中平均等待时间 Wq | Wq = λ/(μ(μ-λ)) | 不包括服务时间 |

**关系**：L = λW, Lq = λWq（Little公式）

### 6.3 例题

某客服中心平均每10分钟到达1个客户（λ=6人/小时），平均服务时间5分钟（μ=12人/小时）。

ρ = 6/12 = 0.5（系统50%时间忙碌）
L = 6/(12-6) = 1人（系统中平均1人）
Lq = 6²/(12×6) = 0.5人（队列中平均0.5人）
W = 1/(12-6) = 1/6小时 = 10分钟（平均逗留10分钟）
Wq = 6/(12×6) = 1/12小时 = 5分钟（平均等待5分钟）

## 七、存储论

### 7.1 什么时候用

题目特征：**进货/存货，求最优订货量使总成本最小**

关键词：订货成本、存储成本、需求率、经济订货量

### 7.2 EOQ模型（经济订货量）

**假设**：需求均匀、瞬时补货、不允许缺货

**核心公式**：

设 D = 年需求量, C₀ = 每次订货成本, Cₕ = 单位年存储成本

| 指标 | 公式 |
|------|------|
| 经济订货量 Q* | Q* = √(2DC₀/Cₕ) |
| 最优订货次数 N* | N* = D/Q* = √(DCₕ/(2C₀)) |
| 最优订货周期 T* | T* = Q*/D（年） |
| 最小总成本 TC* | TC* = √(2DC₀Cₕ) |

### 7.3 例题

某公司年需求量D=1000件，每次订货成本C₀=50元，单位年存储成本Cₕ=4元。

Q* = √(2×1000×50/4) = √25000 ≈ 158件
N* = 1000/158 ≈ 6.3次/年
TC* = √(2×1000×50×4) = √400000 ≈ 632元/年

## 九、考试常见陷阱

**陷阱1：运输问题 vs 指派问题搞混**
- 运输：一个产地可供给多个销地，运量任意 → 伏格尔法
- 指派：一人只做一事，0或1 → 匈牙利法
- 判断标准：看是否"一人一事"

**陷阱2：不确定型 vs 风险型搞混**
- 不确定型：不知道概率 → 5种方法选1种
- 风险型：知道概率 → EMV
- 判断标准：题目有没有给概率

**陷阱3：鞍点法 vs 纳什均衡搞混**
- 鞍点法：二人零和博弈，我赢=你输
- 纳什均衡：非零和博弈，双方可以都赢
- 判断标准：看收益矩阵是单值（零和）还是双值（非零和）

**陷阱4：匈牙利法求最大值直接用**
- ❌ 错误：直接对最大值问题用匈牙利法
- ✅ 正确：先转化为最小值问题（M-原值），再用匈牙利法

**陷阱5：排队论ρ≥1**
- ρ=λ/μ必须<1，否则系统不稳定，队列无限增长
- 如果题目算出ρ≥1，说明服务能力不足，需要增加服务台
