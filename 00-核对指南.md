# MD文件核对指南

> **用途**：本文件指导新会话逐个核对学习笔记MD文件，确保内容与《信息系统项目管理师教程（第四版）》教材原文一致。
>
> **为什么要核对**：之前的AI生成内容中，**所有内容**都可能存在偏差——不仅限于"预期目标""检查方法""绩效要点"，还包括过程名称、ITTO、定义、分类、公式、策略名称等一切内容。AI会"编造"看似合理但教材中根本不存在的说法，照着背一定会翻车。**必须逐字逐句与教材原文核对，不能有任何侥幸心理。**
>
> **核对原则**：
> 1. **一切以教材为准**：MD文件中的每一个知识点都必须与教材原文一致，不允许任何"差不多就行"的模糊表述
> 2. **不能瞎编**：如果教材中没有某个说法，MD文件中就不能出现；如果教材的措辞是A，MD文件就不能写成B
> 3. **在教材基础上总结讲解**：可以有大白话解释、项目实例等辅助理解内容，但核心知识点必须先与教材一致，再进行解释
> 4. **只看这些MD就能通过考试**：这是最终目标——MD文件必须做到完整覆盖教材考点、措辞与教材一致、辅助理解到位
> 5. 宁可多查，不可漏查

## 一、核对方法（每个文件通用）

### 步骤1：读取MD文件
读取目标MD文件的全部内容。

### 步骤2：搜索教材原文
用WebSearch搜索以下关键词组合，找到教材原文：
- `信息系统项目管理师教程第四版 + [绩效域/管理名称] + 预期目标`
- `信息系统项目管理师第四版 + [绩效域/管理名称] + 绩效要点`
- `软考高级 + [绩效域/管理名称] + 检查方法`
- `高项 + [绩效域/管理名称] + 教材原文`

也可以搜索：
- `信息系统项目管理师第四版 + [绩效域/管理名称] + 考点`
- `软考高项 + [绩效域/管理名称] + 笔记`

### 步骤3：逐项比对（全面核对，不放过任何内容）
将MD文件中的**所有内容**与搜索到的教材原文逐项比对。**不要只核对已标注的重点项，所有知识点都要核对！**

#### 绩效域文件必须核对的内容：
- **预期目标**：数量是否正确？措辞是否与教材一字不差？
- **检查方法/指标**：是否与教材原文一致？数量是否正确？
- **绩效要点**：是否与教材结构一致？有没有遗漏或多余？每个要点的名称和内容是否准确？
- **关键定义**：是否与教材一致？
- **分类/层级**：是否与教材的分类方式一致？
- **任何列表/要点**：数量、名称、措辞是否与教材一致？

#### 十大管理文件必须核对的内容：
- **过程名称**：是否与教材一致？数量是否正确？
- **过程组归属**：每个过程属于哪个过程组是否正确？
- **ITTO**（输入、工具与技术、输出）：是否与教材一致？——这个最容易出错，必须逐项核对
- **关键定义**：是否与教材一致？
- **公式**：是否正确？
- **策略/方法名称**：是否与教材一致？（如风险应对策略、冲突管理策略等）
- **任何列表/要点**：数量、名称、措辞是否与教材一致？

#### 通用核对原则：
- 如果教材有N个要点，MD文件也必须是N个，不能多也不能少
- 如果教材的措辞是A，MD文件就不能写成"类似A"的B
- 如果MD文件中有教材里没有的内容，必须删除或标注为"补充理解，非教材原文"
- 如果教材中有MD文件遗漏的内容，必须补充

### 步骤4：修正MD文件
将不一致的内容修正为教材原文。修正规则：
- **核心知识点**（预期目标、检查方法、绩效要点、过程名称、ITTO、定义等）：必须与教材原文完全一致
- **辅助理解内容**（大白话解释、项目实例、记忆技巧等）：保留，但必须标注为辅助理解，不能与教材原文混淆
- **教材没有的内容**：删除，或明确标注"补充理解，非教材原文"
- **教材有但MD遗漏的内容**：必须补充

### 步骤5：标注已核对
修正完成后：
- 在MD文件顶部的元信息区域添加 `> **核对状态**：✅ 已核对（核对日期：YYYY-MM-DD）`
- 在"预期目标"表格标题中标注"（核心！教材原文）"
- 在其他已核对的核心内容处标注"（教材原文）"
