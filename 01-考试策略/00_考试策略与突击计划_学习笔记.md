# 软考高项突击计划与考试策略

## 一、考试概况

| 科目 | 考试时间 | 题型 | 及格线 |
|------|---------|------|--------|
| 综合知识 | 9:00-11:30（150分钟） | 75道单选题 | 45分 |
| 案例分析 | 13:30-15:00（90分钟） | 3道大题（问答+计算） | 45分 |
| 论文 | 15:30-17:30（120分钟） | 2选1，手写2500字 | 45分 |

**重要**：三科同时过才算过，一科不过全部重考！

---

## 二、突击学习计划（假设剩余时间有限）

### 第一阶段：抓大放小（占分60%以上的重点）

**优先级1：十大管理（每天2-3个）**
- 整合、范围、进度、成本、质量、资源、沟通、风险、采购、干系人
- 每个管理领域的：过程数、过程名、过程组归属、核心输入输出工具
- 重点背ITTO速查表中的**核心工具**

**优先级2：八大绩效域（每天1-2个）**
- 干系人、团队、开发方法、规划、项目工作、交付、度量、不确定性
- 重点理解每个绩效域的目标和检查方法

**优先级3：挣值计算（必须拿下！）**
- PV、EV、AC、SV、CV、SPI、CPI、EAC、ETC、VAC
- 每天至少练3道题，直到闭着眼都能算

### 第二阶段：查漏补缺（占分20-30%的内容）

**信息化与技术基础**
- 云计算三种模式（IaaS/PaaS/SaaS）
- 大数据5V
- 物联网三层架构
- 新基建七大领域

**法律法规**
- 招标投标法核心条款
- 政府采购六种方式
- 数据安全法、网络安全法基本概念

### 第三阶段：冲刺模拟（考前3-5天）

- 做3-5套真题，严格计时
- 案例分析要动手写，不要只看
- 论文要准备2-3个模板，背熟开头结尾

## 三、各科目答题策略

### 综合知识（选择题）

**时间分配**：150分钟/75题 = 每题2分钟，留30分钟检查

**答题技巧**：
1. **先易后难**：一眼会的先做，不会的标记跳过
2. **绝对词慎选**：选项有"必须""一定""所有"等绝对词的，大概率错
3. **矛盾选项二选一**：两个选项互相矛盾，正确答案就在其中
4. **计算题不空着**：即使不会，也要蒙一个，写个公式可能有过程分
5. **英文题不放弃**：常见的技术英文缩写要认识（EVM、WBS、PMO等）

**必考点（每年必考）**：
- 挣值计算（3-5分）
- 关键路径计算（2-3分）
- 沟通渠道计算（1分）
- 合同类型选择（1-2分）
- 变更流程（1-2分）
- 风险应对策略（1-2分）
- 冲突管理策略（1分）
- 云计算三种模式（1分）
- 法律法规条款（3-5分）

### 案例分析

**时间分配**：90分钟/3题 = 每题30分钟

**答题技巧**：
1. **先看问题，再看案例**：带着问题读案例，提高效率
2. **找问题要具体**：不要写"管理不好"，要写"没有制定项目章程""变更没有走CCB流程"
3. **计算题要写步骤**：即使结果错了，步骤对也有分
4. **对策要可行**：不要写空话，要写"建立CCB""制定沟通管理计划"等具体措施
5. **分点作答**：用①②③标号，阅卷老师好找得分点

**常见案例类型**：
- **变更失控类**：没走变更流程、没CCB、范围蔓延
- **进度延期类**：关键路径分析、赶工、快速跟进
- **成本超支类**：挣值分析、原因分析、纠偏措施
- **质量差类**：QA/QC不到位、测试不充分
- **团队冲突类**：冲突管理、团队建设
- **沟通不畅类**：沟通计划不完善、干系人管理不到位

**答题模板（找问题类）**：
```
该项目在XX管理方面存在以下问题：
① 没有制定XX计划/XX计划不完善；
② XX过程执行不到位；
③ 缺少XX（如CCB、变更流程、评审机制）；
④ XX（具体描述案例中的问题）；
⑤ 建议：制定XX计划、建立XX机制、加强XX。
```

### 论文

**时间分配**：120分钟，构思10分钟+写作100分钟+检查10分钟

**论文结构（2500字左右）**：

```
摘要（300字）：
- 项目背景（项目名称、规模、工期、投资）
- 项目特点
- 本文论述的XX管理

正文：
1. 项目概述（400字）
   - 项目背景、目标、范围
   - 项目组织结构
   - 技术架构

2. XX管理过程（1500字）
   - 按过程组或过程展开
   - 每个过程写：做了什么、用了什么工具、遇到什么困难、怎么解决的
   - 一定要举具体例子！

3. 项目收尾与总结（300字）
   - 项目成果
   - 经验教训
   - 对XX管理的认识
```

**论文备考策略**：
1. **准备1个项目案例**：准备一个你参与过的真实项目（可以适度虚构），包括：
   - 项目名称、类型（如ERP系统、政务平台、电商平台）
   - 项目金额（建议500-2000万）、工期（6-12个月）
   - 团队规模（10-30人）
   - 技术架构（Java/Spring Cloud/MySQL/Redis等）

2. **准备3-5个论文模板**：
   - 整合管理
   - 范围管理
   - 进度管理
   - 成本管理
   - 风险管理
   - 质量管理

3. **背熟开头结尾**：
   - 开头模板："20XX年X月，我作为项目经理参与了XX项目的建设..."
   - 结尾模板："经过X个月的努力，项目顺利通过验收..."

4. **必写内容**：
   - 项目背景要真实可信
   - 每个过程都要有"我做了什么"
   - 要有具体问题和解决方案
   - 要有工具技术的应用

**论文评分要点**：
- 项目背景真实可信（10分）
- 管理过程完整（30分）
- 有实际案例和问题解决（30分）
- 结构清晰、语言流畅（20分）
- 字迹工整（10分）

## 四、考前必背清单

### 计算题公式（必背！）

**挣值分析**：
- PV = 计划价值
- EV = 挣值
- AC = 实际成本
- SV = EV - PV（>0超前，<0滞后）
- CV = EV - AC（>0节约，<0超支）
- SPI = EV / PV（>1超前，<1滞后）
- CPI = EV / AC（>1节约，<1超支）
- EAC = BAC / CPI（典型偏差）
- EAC = AC + (BAC - EV)（非典型偏差）
- ETC = EAC - AC
- VAC = BAC - EAC

**沟通渠道**：
- 沟通渠道数 = n(n-1)/2

**三点估算**：
- 期望工期 = (乐观 + 4×最可能 + 悲观) / 6
- 标准差 = (悲观 - 乐观) / 6

### 十大管理过程数（必背！）

| 管理领域 | 过程数 | 过程名（口诀） |
|---------|--------|--------------|
| 整合管理 | 7 | 章计指监知控收 |
| 范围管理 | 6 | 规收定分确控 |
| 进度管理 | 6 | 规定排估制控 |
| 成本管理 | 4 | 规估制控 |
| 质量管理 | 3 | 规管控 |
| 资源管理 | 6 | 规估获建管控 |
| 沟通管理 | 3 | 规管监 |
| 风险管理 | 7 | 规识定规规实监 |
| 采购管理 | 4 | 规实控收 |
| 干系人管理 | 4 | 识规管监 |

### 变更流程（必背！）

1. 记录变更请求
2. 评估变更影响（范围、进度、成本、质量）
3. 提交CCB审批
4. 批准后更新计划
5. 执行变更
6. 记录变更结果

## 五、考前注意事项

1. **带齐证件**：身份证、准考证
2. **提前踩点**：熟悉考场位置，避免迟到
3. **论文带笔**：多带几支黑色签字笔，确保书写流畅
4. **注意休息**：考前一天不要熬夜，保持头脑清醒
5. **先易后难**：不要在一道题上死磕，该跳就跳
6. **论文写满**：尽量写满字数，不要留大片空白
7. **字迹工整**：阅卷老师看字给印象分，潦草的字吃亏

---

## 六、最后的话

软考高项不是很难，但内容很多。突击的关键是：
1. **抓重点**：十大管理+八大绩效域+挣值计算，占分超70%
2. **多做题**：真题是最好的复习资料，至少做5套
3. **动手写**：案例分析和论文一定要动手写，不要只看
4. **背模板**：论文准备模板，考试直接套

祝你考试顺利，一次通过！
