# 09 整合管理

> **核对状态**：✅ 已核对（核对日期：2026-05-16）

## 一、一句话理解
整合管理就是"项目经理的本职工作"——把所有零散的管理活动（范围、进度、成本、质量等）串成一条线，确保项目各部分协调一致，最终达成目标。

## 二、核心概念与框架

整合管理是十大管理的"总管家"，负责协调其他九个管理领域，确保项目作为一个整体来推进。

**整合管理的5个目标（教材原文）**：
1. 资源分配
2. 平衡竞争性需求
3. 研究各种备选方法
4. 裁剪过程以实现项目目标
5. 管理各个项目管理知识领域之间的依赖关系

| 过程组 | 过程名称 | 核心作用 |
|--------|---------|---------|
| 启动 | 制定项目章程 | 给项目"发出生证"，正式授权项目经理 |
| 规划 | 制定项目管理计划 | 把所有子计划整合成一份总计划 |
| 执行 | 指导与管理项目工作 | 按计划执行，产出可交付成果 |
| 执行 | 管理项目知识 | 记录经验教训，别让知识随人走 |
| 监控 | 监控项目工作 | 盯着项目进展，发现偏差 |
| 监控 | 实施整体变更控制 | 所有变更统一审批，防止失控 |
| 收尾 | 结束项目或阶段 | 正式收尾，释放资源，归档文档 |

## 三、关键知识点逐个讲

### 知识点1：项目章程
- **什么意思**：项目的"出生证"，由发起人签发，正式授权项目经理动用组织资源。
- **为什么重要**：没有章程，项目经理名不正言不顺，没人听你的。
- **考试怎么考**：选择题常考章程的内容（项目目的、可测量目标及成功标准、高层级需求/项目描述/边界/主要可交付成果、整体项目风险、总体里程碑进度计划、预先批准的财务资源、关键干系人名单、项目审批要求、项目退出标准、委派PM及职责职权、发起人及批准人员的姓名和职权——共11项）。

🏗️ **项目实例**：建工集团CIO在项目启动会上签署了项目章程，写明"授权项目经理调配18人团队、预算审批权限50万元以内、人员调配权"，这就是我的"尚方宝剑"——后来某子公司想绕过我直接给开发团队派活，我拿章程说"所有资源调配必须经过PM"，对方就没话说了。

### 知识点2：项目管理计划
- **什么意思**：项目的"总蓝图"，整合所有子计划（范围、进度、成本、质量、资源、沟通、风险、采购、干系人）。
- **为什么重要**：计划不整合，各管各的，最后拼凑不起来。
- **考试怎么考**：选择题考项目管理计划vs项目文件的区别；案例考计划不完整导致的问题。

🏗️ **项目实例**：我先把范围、进度、成本三个子计划做好，然后发现进度计划里"12家子公司对接"需要资源计划支持，资源计划又影响成本计划——这就是为什么要"整合"：单独做任何一个计划都不完整，必须把所有子计划串起来看有没有矛盾。

### 知识点3：变更控制
- **什么意思**：所有变更都要经过统一审批，评估对范围、进度、成本、质量的影响后再决定。
- **为什么重要**：变更失控是项目失败的首要原因。
- **考试怎么考**：高频考点！变更流程（提出→评估→审批→执行→验证→归档）、CCB（变更控制委员会）的作用。

🏗️ **项目实例**：项目进行到第4个月，物资部突然说"我们还要加一个物料智能预警模块"。我没有直接答应，而是：
1. **记录变更请求**——写在变更日志里
2. **评估影响**——这个模块需要2个开发做3周，进度延后2周，成本增加15万
3. **提交CCB审批**——CCB由集团CIO、信息中心主任、我组成，讨论后决定：加，但放到第二批上线
4. **执行变更**——更新范围基准和进度计划
5. **归档**——变更记录存档

这就是完整的变更流程。如果我当时直接答应，就是"范围蔓延"，项目肯定失控。

### 知识点4：经验教训
- **什么意思**：项目过程中学到的"什么做得好、什么做得差"，记录下来供未来项目参考。
- **为什么重要**：不记录，同样的错误会反复犯。
- **考试怎么考**：案例考知识管理缺失（人走了知识也走了）。

🏗️ **项目实例**：对接第3家子公司时，因为老旧系统接口文档不全，开发团队花了2周才搞定。我让团队把"逆向解析接口的方法"记录到经验教训登记册里。对接第4家时，团队直接用了同样的方法，3天就搞定了——这就是知识管理的价值。

## 五、ITTO串讲（理解式记忆）

**制定项目章程（启动）**
- 它在干什么：给项目"发出生证"，正式授权项目经理。
- 为什么第一个做：没有章程，项目经理没有正式权力。
- 核心输出：项目章程（含高层级目标、预算、里程碑、PM权限）。

**制定项目管理计划（规划）**
- 它在干什么：把所有子计划整合成一份总计划。
- 为什么重要：计划不整合，各管各的，最后拼凑不起来。
- 核心输出：项目管理计划（总蓝图）。

**指导与管理项目工作（执行）**
- 它在干什么：按计划执行，产出可交付成果。
- 核心输出：可交付成果、工作绩效数据（原始记录）、问题日志（首次创建）。

**管理项目知识（执行）**
- 它在干什么：记录经验教训，积累组织过程资产。
- 核心输出：经验教训登记册。

**监控项目工作（监控）**
- 它在干什么：分析工作绩效信息，发现问题和偏差。
- 核心输出：工作绩效报告（给领导看的分析报告）。

**实施整体变更控制（监控）**
- 它在干什么：所有变更统一审批。
- 核心输出：批准的变更请求。
- **变更流程必背**：提出→评估→审批→执行→验证→归档。

**结束项目或阶段（收尾）**
- 它在干什么：正式收尾，移交成果，释放资源。
- 核心输出：最终报告、组织过程资产更新。

## 七、与其他模块的联动

### 与"十大管理其他领域"的联动
整合管理是"总管家"，协调其他九个管理领域。比如：
- 范围变更会影响进度和成本，整合管理要确保变更经过整体评估。
- 进度延误可能需要调整资源或范围，整合管理要协调各方达成一致。

### 与"八大绩效域"的联动
整合管理对应"项目工作绩效域"和"规划绩效域"，是绩效域落地的管理手段。

## 八、常见考试出题点

**选择题挖坑**：
- 项目章程由谁签发？（项目发起人或高层，不是项目经理！）
- 项目管理计划vs项目文件的区别
- 变更流程的顺序和责任人
- 工作绩效数据→信息→报告的数据流

**案例分析怎么考**：
- 变更管理混乱（没有流程、随意变更、范围蔓延）
- 计划不完整（缺少某个子计划）
- 收尾不规范（文档不全、资源没释放）

**论文怎么考**：
- "论信息系统项目的整体管理"——按7个过程写
- 重点写：变更管理、计划整合、经验教训
