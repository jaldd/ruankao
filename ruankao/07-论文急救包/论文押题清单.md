### 重要趋势

- **2023年之前**：只考十大管理
- **2023年下半年开始**：出现绩效域题目（工作绩效域）
- **2024年**：回归十大管理（进度、成本）
- **2025年上半年**：连续两批都考绩效域（交付、不确定性）
- **2025年下半年及以后**：绩效域是绝对重点，十大管理也不能丢

## 二、2026年押题方向（按优先级排序）

### 第一梯队：必准备（概率>70%）

#### 1. 交付绩效域 ⭐⭐⭐⭐⭐

**为什么押它**：2025年第一批刚考过，但子题目非常典型，未来还会变着花样考。

**可能的出题角度**：
- 交付绩效域的预期目标与检查指标
- 可交付物管理与验收
- 价值交付与业务目标对齐
- 质量在交付绩效域中的作用

**必须准备的子题目**：
- 作为项目经理需重点关注哪些方面？（价值交付、可交付物、质量）
- 结合项目说明五大过程组与交付绩效域的协同
- 制定交付绩效域的测量指标
- 如何确保交付物与业务目标一致？

---

#### 2. 团队绩效域 ⭐⭐⭐⭐⭐

**为什么押它**：2025年考了交付和不确定性，团队绩效域是八大绩效域中最容易结合实际项目的，还没考过。

**可能的出题角度**：
- 高绩效团队的特征与建设
- 团队领导力与团队文化
- 团队发展与Tuckman模型
- 团队能力建设与培训

**必须准备的子题目**：
- 如何建设高绩效团队？
- 团队绩效域的预期目标与检查指标
- 结合项目说明团队发展各阶段的管理策略
- 团队绩效域与其他绩效域的协同

---

#### 3. 风险管理 ⭐⭐⭐⭐

**为什么押它**：2023年考过，但风险管理是永恒的热点，且2025年考了不确定性绩效域（与风险相关），说明命题组关注这个方向。

**可能的出题角度**：
- 风险识别与应对策略
- 风险登记册的维护
- 定量风险分析（蒙特卡洛、决策树）
- 储备分析（应急储备vs管理储备）

**必须准备的子题目**：
- 详细阐述风险识别和风险应对
- 风险登记册的内容及如何逐步完善
- 威胁和机会的应对策略
- 储备分析

---

### 第二梯队：重点准备（概率40-70%）

#### 4. 度量绩效域 ⭐⭐⭐⭐

**为什么押它**：2025年交付绩效域考了"制定测量指标"，度量绩效域是绩效域中专门讲指标的，逻辑上最可能考。

**可能的出题角度**：
- 度量指标的确定与对齐
- 挣值分析在度量中的应用
- 度量驱动的持续改进
- 度量绩效域与其他绩效域的协同

---

#### 5. 成本管理 ⭐⭐⭐

**为什么押它**：2024年第二批刚考过，但成本管理是案例和论文的高频考点，挣值分析是必考内容。

**可能的出题角度**：
- 成本估算方法
- 成本基准与储备
- 挣值分析（EVM）
- 成本控制的措施

---

#### 6. 质量管理 ⭐⭐⭐

**为什么押它**：2022年考过，但质量管理工具多、容易写实例，且与交付绩效域紧密相关。

**可能的出题角度**：
- 质量成本（COQ）
- 七种基本质量工具的应用
- 质量保证与质量控制的区别
- 持续改进

---

#### 7. 规划绩效域 ⭐⭐⭐

**为什么押它**：规划是项目的基础，规划绩效域涉及范围、进度、成本等多个方面，容易出题。

**可能的出题角度**：
- 滚动式规划
- 规划与变更的平衡
- 干系人参与规划
- 规划绩效域与其他绩效域的协同

## 三、子题目4大类型与答题套路

### 类型1：理论阐述类

**典型问法**：
- "请阐述XX管理的工具和技术"
- "请论述XX绩效域的核心要点"
- "请说明XX的预期目标"

**答题套路**：定义（50字）+ 核心要素（100字）+ 应用场景（50字）

**示例**（不确定性绩效域）：
```
不确定性绩效域是指与项目不确定性相关的活动和职能，涵盖风险、模糊性、
复杂性三个方面。有效执行该绩效域可实现：了解运行环境、识别应对不确定性、
掌握依赖关系、预测威胁机会、最小化负面影响、利用机会改进、有效利用储备
等预期目标。在我的项目中，我重点关注了接口对接的技术模糊性和多部门协调的
复杂性。
```

---

### 类型2：案例分析类（最常见！）

**典型问法**：
- "请结合你的项目说明"
- "你是如何应用的"
- "请根据你所描述的项目，写出……"

**答题套路**：情境（80字）+ 原因（80字）+ 方案（150字）+ 效果（50字）

**示例**（交付绩效域协同）：
```
在规划阶段，我结合交付绩效域的要求，与规划绩效域协同制定了分阶段交付计划。
由于涉及12个业务部门的协调，交付节奏存在不确定性。我采用"预测+适应"混合
开发模式，设置3个里程碑验收点，并建立变更应急机制。最终各阶段交付物均按时
通过验收，客户满意度达95%以上。
```

---

### 类型3：对比分析类

**典型问法**：
- "请比较XX与XX的区别"
- "各自适用场景是什么"

**答题套路**：相同点（80字）+ 不同点（150字）+ 适用场景（100字）

**示例**（赶工vs快速跟进）：
```
赶工和快速跟进都是进度压缩技术。赶工是增加资源换时间（如加班、加人），
会增加成本；快速跟进是把串行活动变并行（如设计和开发同时进行），会增加
风险。赶工适用于有资源冗余的情况，快速跟进适用于活动之间依赖关系可调整的
情况。在我的项目中，某模块延期时，我先用快速跟进把测试提前介入，同时赶工
增加了2名开发人员，双管齐下抢回了进度。
```

---

### 类型4：改进建议类

**典型问法**：
- "如何改进"
- "有哪些措施"
- "应该怎么做"

**答题套路**：现状问题（80字）+ 改进措施（200字）+ 预期效果（50字）

**示例**：
```
项目初期存在需求确认周期过长的问题。我采取了三项改进措施：
① 建立需求评审机制，每周固定原型评审会；
② 采用敏捷需求池管理，优先级高的优先开发；
③ 设置需求签字确认流程，明确验收标准。
改进后需求确认周期从15天缩短至8天，需求变更率下降40%。
```

## 五、万能项目背景的适配性

用同一个政务大数据平台项目（1200万/28人/12个月），所有论文都能套：

| 论文题目 | 项目特点怎么套 |
|---------|--------------|
| 交付绩效域 | 12个部门的数据整合，交付物多、验收复杂 |
| 团队绩效域 | 28人团队，跨职能协作，需要高绩效团队 |
| 风险管理 | 接口对接多、政策法规变化、技术不确定性 |
| 度量绩效域 | 数据驱动的项目管理，SPI/CPI/缺陷率等指标 |
| 成本管理 | 1200万预算，多供应商，成本控制压力大 |
| 质量管理 | 政务系统质量要求高，可用性>99.9% |
| 整合管理 | 多部门协调，变更频繁，需要整体把控 |
| 范围管理 | 12个部门需求不同，范围界定和变更控制难 |

## 七、最后的话

论文及格的关键不是写得有多好，而是：
1. **不跑题**——子题目全部回应
2. **有实例**——每个理论点都有项目案例
3. **结构清**——摘要+过渡+过程+总结，层次分明
4. **字数够**——2500字左右，不要太少

押题只是提高命中率，真正保险的是**多准备几篇、多动手写**。祝你论文一次过！
