# 软考高项选择题高频考点（考前背诵版）

## 使用说明
- 本文档按考点分类整理，每类标注**考试频率**和**分值预估**
- 建议考前两天每天过一遍，重点看**五星考点**
- 记忆口诀用【】标注，方便快速记忆

| 管理领域 | 过程数 | 过程名口诀 |
|---------|--------|-----------|
| 整合管理 | 7 | 章计指监知变收 |
| 范围管理 | 6 | 规收定分确控 |
| 进度管理 | 6 | 规定排估制控 |
| 成本管理 | 4 | 规估制控 |
| 质量管理 | 3 | 规管控 |
| 资源管理 | 6 | 规估获建管控 |
| 沟通管理 | 3 | 规管监 |
| 风险管理 | 7 | 规识定规规实监 |
| 采购管理 | 4 | 规实控收 |
| 干系人管理 | 4 | 识规管监 |

【记忆口诀】：整7范6进6成4质3资6沟3风7采4干4

---

### 1.2 五大过程组与十大管理矩阵（高频！）⭐⭐⭐⭐⭐

| 过程组 | 包含的过程 |
|--------|-----------|
| 启动 | 制定项目章程、识别干系人 |
| 规划 | 大部分规划过程（制定XX计划） |
| 执行 | 指导与管理项目工作、管理项目知识、建设团队、管理团队、管理沟通、实施风险应对、实施采购、管理干系人参与 |
| 监控 | 监控项目工作、实施整体变更控制、确认范围、控制范围、控制进度、控制成本、控制质量、控制资源、监督沟通、监督风险、控制采购、监督干系人参与 |
| 收尾 | 结束项目或阶段 |

【记忆技巧】：启动2个、收尾1个，其余大部分在规划和监控。

---

### 1.3 变更流程（每年必考！）⭐⭐⭐⭐⭐

1. 记录变更请求
2. 评估变更影响（范围、进度、成本、质量）
3. 提交CCB审批
4. 批准后更新计划
5. 执行变更
6. 记录变更结果

【口诀】：记评批执归（记录-评估-审批-执行-归档）

---

### 1.4 项目章程 vs 项目管理计划（高频！）⭐⭐⭐⭐

| 对比项 | 项目章程 | 项目管理计划 |
|--------|---------|-------------|
| 作用 | 给项目"发出生证" | 项目的"总蓝图" |
| 签发人 | 项目发起人或高层 | 项目经理制定，高层批准 |
| 内容 | 高层级目标、预算、里程碑、PM权限 | 所有子计划的整合 |
| 修改 | 很少修改 | 可更新 |

## 二、计算题考点（每年必考，占10-15分）

### 2.1 挣值分析（必考！）⭐⭐⭐⭐⭐

| 指标 | 公式 | 判断 |
|------|------|------|
| SV | EV - PV | >0超前，<0滞后 |
| CV | EV - AC | >0节约，<0超支 |
| SPI | EV / PV | >1超前，<1滞后 |
| CPI | EV / AC | >1节约，<1超支 |
| EAC | BAC/CPI（典型）或AC+(BAC-EV)（非典型） | - |
| ETC | EAC - AC | - |
| VAC | BAC - EAC | - |

【口诀】：EV在前都是好的，SPI看进度，CPI看成本，>1好事<1坏事

---

### 2.2 沟通渠道（每年1分）⭐⭐⭐⭐

公式：沟通渠道数 = n(n-1)/2

【例子】：5人团队，渠道数=5×4/2=10；增加1人变成6人，渠道数=6×5/2=15，增加50%！

---

### 2.3 三点估算（每年1-2分）⭐⭐⭐⭐

- 期望工期 = (乐观 + 4×最可能 + 悲观) / 6
- 标准差 = (悲观 - 乐观) / 6

---

### 2.4 关键路径（每年2-3分）⭐⭐⭐⭐

- 关键路径：总浮动时间=0的活动序列
- 关键路径决定项目最短工期
- 关键路径上的活动延误会导致项目延期

## 三、质量管理（每年3-5分）⭐⭐⭐⭐⭐

### 3.1 七种基本质量工具

| 工具 | 用途 | 口诀 |
|------|------|------|
| 因果图（鱼骨图） | 找根因 | 鱼骨找根因 |
| 流程图 | 看步骤 | 流程看步骤 |
| 核查表 | 做统计 | 核查做统计 |
| 帕累托图 | 找重点（80/20） | 帕累托找重点 |
| 直方图 | 看分布 | 直方看分布 |
| 控制图 | 看稳定 | 控制看稳定 |
| 散点图 | 看关系 | 散点看关系 |

### 3.2 质量成本（COQ）

| 类型 | 说明 | 例子 |
|------|------|------|
| 预防成本 | 防止缺陷 | 培训、流程设计 |
| 评估成本 | 评估是否符合 | 测试、检查 |
| 内部失败成本 | 交付前发现 | 返工、废品 |
| 外部失败成本 | 交付后发现 | 保修、投诉 |

【口诀】：预防评估是投资，内部外部是损失

### 3.3 管理质量 vs 控制质量

| 对比 | 管理质量（QA） | 控制质量（QC） |
|------|---------------|---------------|
| 关注点 | 过程 | 产品 |
| 目的 | 预防缺陷 | 发现缺陷 |
| 工具 | 质量审计、过程分析 | 检查、测试、控制图 |

## 四、风险管理（每年3-5分）⭐⭐⭐⭐⭐

### 4.1 威胁应对策略

| 策略 | 说明 | 例子 |
|------|------|------|
| 规避 | 让风险不发生 | 取消高风险功能 |
| 转移 | 把风险给别人 | 买保险、外包 |
| 减轻 | 降低概率或影响 | 增加测试、备份 |
| 接受 | 认了，不采取措施 | 建立应急储备 |

### 4.2 机会应对策略

| 策略 | 说明 |
|------|------|
| 开拓 | 确保机会发生 |
| 分享 | 和别人一起抓 |
| 提高 | 增加发生概率 |
| 接受 | 不主动追求 |

### 4.3 储备分析

| 储备类型 | 应对 | 谁有权用 | 是否计入基准 |
|---------|------|---------|-------------|
| 应急储备 | 已知-未知 | 项目经理 | 是（成本基准） |
| 管理储备 | 未知-未知 | 需高层批准 | 否（项目预算） |

## 五、资源管理（每年2-3分）⭐⭐⭐⭐

### 5.1 RACI矩阵

| 角色 | 说明 |
|------|------|
| R（Responsible） | 负责执行的人 |
| A（Accountable） | 最终负责的人（只能有一个） |
| C（Consulted） | 需要咨询的人 |
| I（Informed） | 需要知情的人 |

### 5.2 Tuckman团队五阶段

1. 形成期（Forming）—— 互相认识，依赖领导
2. 震荡期（Storming）—— 冲突出现，争夺地位
3. 规范期（Norming）—— 建立规则，开始协作
4. 成熟期（Performing）—— 高效协作，自我驱动
5. 解散期（Adjourning）—— 项目结束，团队解散

【口诀】：形震规成散

### 5.3 冲突管理五策略

1. 撤退/回避 —— 暂时不处理
2. 缓和/包容 —— 强调一致，淡化分歧
3. 妥协/调解 —— 各让一步
4. 强迫/命令 —— 强行推行一方意见
5. 合作/解决问题 —— 双赢，最优策略

【口诀】：退缓妥强合（最优是合作）

## 十一、答题技巧

### 11.1 选择题答题技巧

1. **先易后难**—— 一眼会的先做，不会的标记跳过
2. **绝对词慎选**—— "必须""一定""所有"等绝对词大概率错
3. **矛盾选项二选一**—— 两个选项互相矛盾，正确答案就在其中
4. **计算题不空着**—— 即使不会，也要蒙一个
5. **英文题不放弃**—— EVM、WBS、PMO等常见缩写要认识

### 11.2 常见陷阱

- "以下哪个**不属于**"—— 注意"不"字
- "以下哪个**最先**"—— 注意顺序
- "以下哪个**最重要**"—— 注意程度
- "项目经理**应该**"—— 注意职责
