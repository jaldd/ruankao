# 04 规划绩效域

> **核对状态**：✅ 已核对（核对日期：2026-05-16）

## 一、一句话理解
规划绩效域涉及整个项目期间组织与协调相关活动职能，这些活动和职能是最终交付项目和成果所必需的。简单说就是"画好地图再出发"——组织好项目的规划工作，让项目有条理地推进。

## 二、管不好会出什么乱子
1. **项目推进混乱**：没有整体规划，各干各的，进度、成本、资源对不上号。
2. **规划太粗或太细**：预测型项目规划太粗，适应型项目规划太细，都不对。
3. **规划不能适应变化**：环境变了计划不能跟着调，一变就乱。

## 三、预期目标与指标对照（核心！教材原文）

> **🧭 三者关系解读**：绩效要点（你怎么做）→ 预期目标（你要什么结果）→ 指标（你怎么量）。三者是**路径→终点→尺子**的因果链，不是三个独立的东西。下表的"预期目标"就是第四节"绩效要点"要达成的结果，"指标"就是检测目标达没达成的尺子。
>
> **本域映射**：规划绩效域是8大绩效域中要点最多(8个)、目标最多(6个)的域之一。8个要点是"规划要做什么事"，6个目标是"规划做好了长什么样"。不是一一对应，而是多个要点共同支撑多个目标。
>
> ```
> 要点1：影响因素 ──→ 决定规划方式（支撑所有目标）
> 要点2：项目估算 ──→ ①有条理推进 ←尺子：绩效偏差
> 要点3：团队规划 ──┐
> 要点4：沟通规划 ──┼──→ ②系统方法交付 ←尺子：规划整体性
> 要点5：实物资源 ──┤──→ ③演变详细说明 ←尺子：规划详尽程度
> 要点6：采购规划 ──┤──→ ④时间成本适当 ←尺子：规划适宜性
> 要点7：变更规划 ──┤──→ ⑤内容充分管理干系人 ←尺子：规划充分性
> 要点8：度量一致性 ┘──→ ⑥可适应变化 ←尺子：可适应变化
> ```

| 预期目标（教材原文） | 大白话解释 | 检查方法/指标（教材原文） | 做得好 | 搞砸了 |
|---------|-----------|-------------|--------|--------|
| ①项目以有条理、协调一致的方式推进 | 项目推进有条理、不混乱 | **绩效偏差**：对照项目基准和其他度量指标对项目结果进行绩效审查表明项目正在按计划进行，绩效偏差处于临界值范围内 | SPI/CPI在临界值内，进度成本可控 | 项目到处救火，进度成本偏差超红线 |
| ②应用系统的方法交付项目成果 | 用系统方法交付，不是东一榔头西一棒 | **规划的整体性**：交付进度、资金提供、资源可用性、采购等表明项目是以整体方式进行规划的，没有差距或不一致之处 | 各子计划之间不矛盾，进度、资金、资源、采购对得上号 | 进度计划说3个月，采购计划说5个月才到货 |
| ③对演变情况进行详细说明 | 对演变情况有详细说明，信息越来越清晰 | **规划的详尽程度**：与当前信息相比，可交付物和需求的初步信息是适当的、详尽的；与可行性研究与评估相比，当前信息表明项目可以生成预期的可交付物和成果 | 规划信息随项目推进越来越详尽，渐进明细 | 规划一直停留在高层级，细节永远不清楚 |
| ④规划投入的时间成本是适当的 | 规划投入的时间成本合适，不多不少 | **规划适宜性**：项目计划和文件表明规划水平适合于项目 | 预测型项目前期规划充分，适应型项目滚动规划适度 | 预测型项目规划太粗，适应型项目规划太细 |
| ⑤规划的内容对管理干系人的需求而言是充分的 | 规划内容能充分管理干系人需求 | **规划的充分性**：沟通管理计划和干系人信息表明沟通足以满足干系人的期望 | 干系人通过沟通计划能获得所需信息，没有信息缺口 | 干系人抱怨"不知道项目进展""没收到该收的报告" |
| ⑥可以根据新出现的和不断变化的需求进行调整 | 能适应变化，不是死守计划不变 | **可适应变化**：采用待办事项列表的项目，在整个项目期间会对各个计划做出调整。采用变更控制过程的项目具有变更控制委员会，会议的变更日志和文档表明变更控制过程正在得到应用 | 适应型项目待办列表持续调整优先级；预测型项目变更走CCB流程 | 变更没人管，想改就改；或变更流程形同虚设 |

**⚠️ 考试重点**：选择题考"预期目标"原文（必须一字不差）；论文考"检查方法/指标"（要会结合实际项目写）！

## 四、绩效要点逐个讲（教材原文：8个绩效要点）

### 要点1：规划的影响因素（教材原文）
规划的影响因素包括：

| 影响因素 | 教材描述 | 大白话解释 |
|---------|---------|-----------|
| 开发方法 | 预测型开发方法大部分规划预先进行；适应型方法一些规划提前进行，进一步规划在每个迭代开始时进行；也可预先进行高层级规划，随后使用原型方法进行设计 | 方法不同，规划方式不同 |
| 项目可交付物 | 产品开发或高技术项目采取持续性和适应性的规划，根据干系人的反馈和技术进步进行演变和变更 | 交付物不同，规划节奏不同 |
| 组织需求 | 组织治理、政策、流程和文化会要求项目经理提供特定的规划成果 | 组织有规矩，规划要遵守 |
| 市场条件 | 市场竞争激烈，为了使产品尽快进入市场，可以进行最低限度的前期规划 | 抢市场可以少规划 |
| 法律或法规的限制 | 监管机构或法律法规有时要求先提供特定的规划文件，然后才能授权实施，或者获得批准向市场发布项目可交付成果 | 法规要求必须先规划 |

🏗️ **项目实例**：我的数据治理项目受以下因素影响：
- **开发方法**：混合型→前期预测型规划数据标准，后期适应型规划业务模块
- **组织需求**：公司要求提供《项目管理计划》《质量管理计划》等特定规划成果
- **市场条件**：集团要求年底前上线，前期规划时间有限，采用最低限度前期规划+滚动式规划

### 要点2：项目估算（教材原文）
规划时需要对工作投入、持续时间、成本、人员和实物资源进行估算。估算是对某一变量（如项目成本、资源、人力投入或持续时间）的可能数值或结果的定量评估。

#### 影响估算的因素
- **区间**：开始寻找项目机会时估算区间为-25%到75%；在项目生命周期进展中估算区间逐步缩小，进展良好项目估算区间一般为-5%到10%
- **准确度**：准确度与区间相关，准确度越低，估算值的潜在区间就越大，随着项目进行项目的准确度逐渐提高
- **信心**：信心会随着经验的增长而增加，处理以前的类似项目经验都有助于提高信心，面对新的和不断演变的技术，估算的信心会降低

#### 估算时可采用的方法
- **确定性估算和概率估算**：确定性估算也称为点估算，表示为一个数字或金额；概率估算包括一定区间内的估算以及该区间内的相关概率。概率估算三个因素：具有一定的区间的点估算、置信程度、概率分布
- **绝对估算和相对估算**：绝对估算是具体信息，使用实际数字；相对估算一般是基于一个特定基准进行
- **基于工作流的估算**：通过确定周期和产量来制定，周期是一个产品经过一个完成过程的总消耗时间，产量是指可以在给定时间内完成一个完整过程的产品数
- **对不确定性的调整估算**：估算本身具有不确定性，不确定性与风险有关，可根据不确定性模拟的结果，通过增加应急储备的方式，调整关键可交付物

🏗️ **项目实例**：项目初期估算区间较大（-25%~75%），随着需求明确逐步缩小到-5%~10%。我用自下而上估算法，把每个WBS工作包的成本逐个估算后汇总，总预算1080万。对不确定性较高的"老旧系统对接"部分，增加了应急储备。

### 要点3：项目团队组成和结构规划（教材原文）
规划项目团队的组成和结构时，充分考虑完成项目所需的技能组合，包括技能、熟练程度和类似项目经验。

🏗️ **项目实例**：我做了资源日历——开发团队18人，但其中2名需求分析师在需求阶段结束后要转岗支持测试；达梦厂商技术专家第3-4个月驻场支持信创适配；测试团队第7个月才全力投入。资源日历帮我避免了资源冲突。

### 要点4：沟通规划（教材原文）
沟通是争取干系人有效参与的最重要的因素，对沟通进行规划时，需要与干系人绩效域进行关联，包括干系人识别、分析、优先级排序和参与的内容。

🏗️ **项目实例**：我制定了沟通管理计划：每周给CIO发周报、每月给信息中心主任做月度汇报、每两周给业务部门做迭代演示、每日站会同步团队进度。

### 要点5：实物资源规划（教材原文）
实物资源指人力资源以外的任何资源，包括材料、设备、软件、测试环境、许可证等。对资源交付、移动、存储和处置进行规划，包括对实物资源的流动轨迹进行跟踪。

🏗️ **项目实例**：我规划了实物资源：达梦数据库许可证（第2个月采购）、测试服务器（第6个月到位）、信创适配环境（第3-4个月搭建）。

### 要点6：采购规划（教材原文）
采购可以在项目期间的任何时候进行，预先规划有助于明确目标，确保采购过程顺利进行。

🏗️ **项目实例**：我提前规划了采购：达梦数据库许可证、信创服务器、数据集成ETL工具。采购周期长的提前3个月启动，避免到用时还没到货。

### 要点7：变更规划（教材原文）
项目团队应对变更控制进行规划，并制定相应的变更管理流程，以便整个项目期间可以按照变更控制流程、重新确定待办事项列表的优先级排序，或者重新确定项目基准，签订合同的项目应遵循已定义的合同变更流程。

🏗️ **项目实例**：我制定了变更管理流程：提出变更→评估影响→CCB审批→执行变更→记录结果。适应型部分用待办事项列表调整优先级，预测型部分走变更控制流程。

### 要点8：度量指标和一致性（教材原文）
- **度量指标**：确定度量指标、基准和临界值，以及确定测试和评估方法及流程是规划绩效域的重要工作
- **一致性**：规划需要保持一致性，各子计划之间不能矛盾

🏗️ **项目实例**：我在规划阶段就确定了度量指标：进度偏差≤5%、成本偏差≤10%、缺陷密度≤5个/千行代码、客户满意度≥90%。这些指标与项目基准和临界值一起，构成了度量的基础。

## 五、具体怎么落地

### 场景1：新项目启动后的规划
**做什么**：项目经理组织核心团队，根据开发方法确定规划方式，进行估算，制定各子计划。
**写什么文件**：《项目管理计划》（总计划）+ 各子计划。
**用什么工具**：WBS分解、关键路径法（CPM）、三点估算、风险登记册。

### 场景2：适应型项目的滚动式规划
**做什么**：前期做高层规划（路线图），每个迭代前做详细规划（迭代计划），边做边调整。
**写什么文件**：《产品路线图》《迭代计划》《发布计划》。
**开什么会**：迭代计划会（规划这迭代做什么）、每日站会（同步进度）。

### 场景3：计划需要重大调整
**做什么**：外部环境变化（如政策调整、技术变革），原计划不再适用，启动变更流程。
**写什么文件**：变更请求、更新后的《项目管理计划》、变更日志。
**走什么流程**：提出变更→评估影响→CCB审批→执行变更→记录结果。

## 六、和其他绩效域的联动

### 与"开发方法和生命周期绩效域"的联动
**场景**：开发方法决定了规划的粒度和方式——预测型需要详细的前期规划，适应型是滚动式规划。
**影响**：方法选错，规划要么太粗要么太细。
**处理**：根据开发方法调整规划策略。

### 与"度量绩效域"的联动
**场景**：规划时要定义度量指标、基准和临界值，度量时要对照计划检查偏差。
**影响**：规划没定义度量指标，度量就无从谈起。
**处理**：规划阶段就确定度量指标和基准。

### 与"干系人绩效域"的联动
**场景**：沟通规划需要与干系人绩效域关联，包括干系人识别、分析、优先级排序和参与的内容。
**影响**：干系人参与不足会导致规划不充分。
**处理**：规划时充分考虑干系人需求。

## 七、涉及的文件和工具

| 文件/工具 | 一句话说明 |
|----------|-----------|
| 项目管理计划 | "总蓝图"，整合所有子计划的综合性文件 |
| WBS | "工作分解表"，把项目目标逐层分解到可管理的工作包 |
| 进度网络图 | "路线图"，显示活动之间的依赖关系 |
| 关键路径 | "瓶颈线"，决定项目最短工期的活动序列 |
| 风险登记册 | "风险清单"，记录所有已识别的风险及应对措施 |
| 变更管理计划 | "变更规矩"，规定变更怎么提、怎么审、怎么批 |
| 基准（范围/进度/成本） | "标尺"，用来衡量项目绩效的参照标准 |

## 八、容易混淆的概念

| 容易混淆的 | 区别 |
|-----------|------|
| 规划绩效域 vs 十大管理的规划过程 | 规划绩效域关注"规划工作本身怎么做好"（影响因素、估算、一致性等）；十大管理的规划过程是"具体规划范围/进度/成本等" |
| 项目管理计划 vs 项目文件 | 项目管理计划是"计划"（规划怎么做）；项目文件是"记录"（记录实际情况） |
| 基准 vs 计划 | 基准是经过批准的"冻结版"计划，变更要走流程；计划是可以调整的 |
| 滚动式规划 vs 渐进明细 | 滚动式规划是方法（近期详细、远期粗略）；渐进明细是特征（信息随时间越来越清晰） |
| 确定性估算 vs 概率估算 | 确定性估算是点估算（一个数字）；概率估算包含区间和概率 |
| 绝对估算 vs 相对估算 | 绝对估算用实际数字；相对估算基于特定基准 |

## 九、常见考试出题点

**选择题挖坑**：
- 规划绩效域6个预期目标（高频考点！注意措辞与教材一致）
- 规划的影响因素5个（开发方法、项目可交付物、组织需求、市场条件、法律或法规限制）
- 估算方法4种（确定性/概率、绝对/相对、基于工作流、对不确定性调整）
- 估算区间：初期-25%~75%，进展良好-5%~10%
- 检查方法6个指标（绩效偏差、规划的整体性、规划的详尽程度、规划适宜性、规划的充分性、可适应变化）

**案例分析怎么考**：
- 规划不合理导致的问题（范围蔓延、进度延期、成本超支）
- 变更管理混乱（没有流程、随意变更、变更影响没评估）
- 计算题：关键路径、挣值分析

**论文怎么考**：
- "论信息系统项目的规划管理"——按规划绩效域的绩效要点写
- **论文常要求写测量指标**，记住第六节的指标表！

## 十、记忆口诀

**规划绩效域6个预期目标**：
"有条系统说演变，适当充分可调整"——①有条理协调一致、②系统方法、③演变详细说明、④时间成本适当、⑤内容充分、⑥可适应变化

**规划影响因素5个**：
"发可组市法"——开发方法、项目可交付物、组织需求、市场条件、法律或法规限制

**估算方法4种**：
"确概绝相工不确"——确定性/概率估算、绝对/相对估算、基于工作流估算、对不确定性调整估算

**6个检查指标**：
"偏整详适充变"——绩效偏差、规划的整体性、规划的详尽程度、规划适宜性、规划的充分性、可适应变化
