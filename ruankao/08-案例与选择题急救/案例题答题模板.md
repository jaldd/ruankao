## 二、常见案例类型与答题模板

### 类型1：找问题/分析原因类（最常考！）

**问法**：
- "请分析该项目在XX管理方面存在的问题"
- "请指出项目经理在XX方面的不足"
- "导致项目失败/延期/超支的原因是什么"

**万能答题模板**：
```
该项目在XX管理方面存在以下问题：
① 没有制定XX计划/XX计划不完善；
② XX过程执行不到位，缺少XX（如评审、确认、跟踪）；
③ 缺少XX机制（如CCB、变更流程、沟通机制）；
④ XX（具体描述案例中的问题，如"需求未经确认就开发"）；
⑤ XX（另一个具体问题）；
⑥ 建议：制定XX计划、建立XX机制、加强XX。
```

**示例**（范围管理问题）：
```
该项目在范围管理方面存在以下问题：
① 没有制定范围管理计划，范围定义不清晰；
② 需求收集不全面，缺少需求确认环节；
③ 没有建立变更控制委员会（CCB），变更随意；
④ 客户频繁加需求，项目经理直接答应，导致范围蔓延；
⑤ WBS分解不合理，部分工作包遗漏；
⑥ 建议：制定范围管理计划、建立CCB、严格执行变更流程。
```

### 类型3：计算题（必考！必须拿下）

#### 3.1 挣值分析计算

**必背公式**：
| 指标 | 公式 | 判断标准 |
|------|------|---------|
| PV | 计划价值 | - |
| EV | 挣值 | - |
| AC | 实际成本 | - |
| SV | EV - PV | >0超前，<0滞后 |
| CV | EV - AC | >0节约，<0超支 |
| SPI | EV / PV | >1超前，<1滞后 |
| CPI | EV / AC | >1节约，<1超支 |
| EAC（典型） | BAC / CPI | - |
| EAC（非典型） | AC + (BAC - EV) | - |
| ETC | EAC - AC | - |
| VAC | BAC - EAC | - |
| TCPI | (BAC - EV) / (BAC - AC) | >1难完成，<1易完成 |

**答题步骤**：
```
第一步：列出已知条件（PV、EV、AC、BAC）
第二步：计算SV、CV、SPI、CPI
第三步：判断项目状态（进度超前/滞后，成本节约/超支）
第四步：计算EAC、ETC、VAC（如题目要求）
第五步：给出结论和建议
```

**示例**：
```
已知：项目总预算BAC=100万元，第6个月末PV=60万，EV=54万，AC=58万。

计算：
SV = EV - PV = 54 - 60 = -6万元（进度滞后）
CV = EV - AC = 54 - 58 = -4万元（成本超支）
SPI = EV / PV = 54 / 60 = 0.9（进度滞后10%）
CPI = EV / AC = 54 / 58 = 0.93（成本超支7%）

EAC（典型偏差）= BAC / CPI = 100 / 0.93 = 107.5万元
ETC = EAC - AC = 107.5 - 58 = 49.5万元
VAC = BAC - EAC = 100 - 107.5 = -7.5万元（预计超支7.5万）

结论：项目目前进度滞后、成本超支，需采取纠偏措施。
建议：① 分析成本超支原因，优化资源分配；② 对剩余工作重新估算，必要时申请追加预算。
```

#### 3.2 关键路径计算

**答题步骤**：
```
第一步：根据活动列表画出网络图（或列出依赖关系）
第二步：计算每个活动的最早开始时间（ES）和最早完成时间（EF）
第三步：计算每个活动的最晚开始时间（LS）和最晚完成时间（LF）
第四步：计算总浮动时间（TF = LS - ES = LF - EF）
第五步：找出总浮动时间为0的活动，即关键路径
第六步：计算项目总工期
```

**示例**：
```
活动A（3天）→ 活动B（5天）→ 活动D（4天）
           ↘ 活动C（2天）↗

计算：
路径1：A-B-D = 3+5+4 = 12天
路径2：A-C-D = 3+2+4 = 9天

关键路径：A-B-D，总工期12天。
活动C的总浮动时间 = 12 - 9 = 3天。
```

#### 3.3 沟通渠道计算

**公式**：沟通渠道数 = n(n-1)/2

**示例**：
```
项目团队有10人，沟通渠道数 = 10×9/2 = 45条。
增加2人后，团队12人，沟通渠道数 = 12×11/2 = 66条。
沟通渠道增加了66-45=21条，增长率为21/45=46.7%。
```

---

#### 3.4 三点估算

**公式**：
- 期望工期 = (乐观 + 4×最可能 + 悲观) / 6
- 标准差 = (悲观 - 乐观) / 6

**示例**：
```
某活动乐观估计6天，最可能估计9天，悲观估计18天。
期望工期 = (6 + 4×9 + 18) / 6 = (6+36+18)/6 = 60/6 = 10天
标准差 = (18 - 6) / 6 = 12/6 = 2天

在10±2天（即8-12天）内完成的概率约68%。
在10±4天（即6-14天）内完成的概率约95%。
```

## 三、案例题高频考点速查

### 3.1 整合管理
- **常见问题**：没有项目章程、计划不完整、变更失控、收尾不规范
- **答题关键词**：项目章程、项目管理计划、CCB、变更流程、经验教训

### 3.2 范围管理
- **常见问题**：范围蔓延、镀金、需求不明确、WBS不合理
- **答题关键词**：范围说明书、WBS、需求跟踪矩阵、范围基准、CCB

### 3.3 进度管理
- **常见问题**：进度延期、关键路径延误、资源冲突
- **答题关键词**：关键路径、赶工、快速跟进、资源平衡、SPI

### 3.4 成本管理
- **常见问题**：成本超支、预算不合理、储备不足
- **答题关键词**：成本基准、应急储备、管理储备、CPI、EAC

### 3.5 质量管理
- **常见问题**：质量差、测试不充分、QA缺失
- **答题关键词**：质量审计、控制图、帕累托图、质量成本、QA/QC

### 3.6 资源管理
- **常见问题**：人员不足、团队冲突、技能不匹配
- **答题关键词**：RACI、Tuckman模型、冲突管理、虚拟团队

### 3.7 沟通管理
- **常见问题**：信息传递不畅、干系人不满
- **答题关键词**：沟通渠道、沟通方式、沟通管理计划

### 3.8 风险管理
- **常见问题**：风险识别不全、应对措施缺失
- **答题关键词**：风险登记册、概率影响矩阵、应对策略、储备

### 3.9 采购管理
- **常见问题**：合同类型选择不当、供应商管理不善
- **答题关键词**：合同类型、SOW、招投标、索赔

### 3.10 干系人管理
- **常见问题**：干系人识别不全、参与度管理不到位
- **答题关键词**：权力/利益方格、参与度评估矩阵、干系人登记册

## 四、案例题答题技巧

### 4.1 读题技巧
1. **先看问题，再看案例**——带着问题读案例，提高效率
2. **标记关键词**——用笔圈出"延期""超支""变更""冲突"等信号词
3. **数问题数量**——确保每个小问都回答到

### 4.2 答题技巧
1. **分点作答**——用①②③标号，阅卷老师好找得分点
2. **先写结论，再写理由**——结论在前，分析在后
3. **计算题写步骤**——即使结果错了，步骤对也有分
4. **不会也要蒙**——写相关的知识点，可能有过程分

### 4.3 常见丢分点
1. **只写问题不写建议**——题目问"怎么办"，必须给方案
2. **术语写错**——"CCB"写成"CBB"，"WBS"写成"WBC"
3. **漏答小问**——一道大题有3-4个小问，漏一个扣5-8分
4. **字迹潦草**——机考就忽略这条

## 五、考前必背清单（案例题专用）

### 5.1 变更流程（必背！）
1. 记录变更请求
2. 评估变更影响（范围、进度、成本、质量）
3. 提交CCB审批
4. 批准后更新计划
5. 执行变更
6. 记录变更结果

### 5.2 风险应对策略（必背！）
- **威胁**：规避、转移、减轻、接受
- **机会**：开拓、分享、提高、接受

### 5.3 冲突管理策略（必背！）
1. 撤退/回避
2. 缓和/包容
3. 妥协/调解
4. 强迫/命令
5. 合作/解决问题（最优）

### 5.4 合同类型选择（必背！）
- 范围明确 → 固定总价
- 范围不明 → 成本补偿
- 灵活用工 → 工料合同

### 5.5 质量七种工具（必背！）
- 鱼骨图（找根因）
- 流程图（看步骤）
- 核查表（做统计）
- 帕累托图（找重点）
- 直方图（看分布）
- 控制图（看稳定）
- 散点图（看关系）

## 六、最后叮嘱

案例题的核心是**"找问题+给方案"**，记住这个公式：
```
案例题得分 = 找全问题（40%）+ 分析准确（30%）+ 方案可行（30%）
```

多练真题，多动手写，不要只看！
