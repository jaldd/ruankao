### ⚠️ ITTO三大易错点

1. **"商业文件" vs "立项管理文件"**：第四版教材中，制定项目章程、规划采购管理、识别干系人的输入是**立项管理文件**，不是商业文件！
2. **"项目文件更新"**：很多过程的输出都包含"项目文件更新"，别漏掉
3. **"组织过程资产"和"事业环境因素"**：几乎所有过程的输入都包含这两个，记住它们是"万能输入"

## 三、八大绩效域——必背核心考点（8-12分）

### 每个绩效域只需记住3件事：目标、检查方法、绩效要点

| 绩效域 | 核心目标 | 必背检查方法 | 必背绩效要点 |
|--------|---------|-------------|-------------|
| 干系人 | 干系人满意 | 干系人满意度、参与度 | 尽早识别、分类管理、持续沟通 |
| 团队 | 共享责任、高绩效 | 团队满意度、技能提升 | 塔克曼五阶段：形成→震荡→规范→成熟→解散 |
| 开发方法与生命周期 | 合适的开发方法 | 交付节奏、开发方法适宜性 | 预测型→适应型，选对方法 |
| 规划 | 有计划、能执行 | 计划完整性、变更频率 | 渐进明细，滚动式规划 |
| 项目工作 | 高效执行 | 工作效率、资源利用 | 持续改进，消除浪费 |
| 交付 | 达到质量要求 | 交付物质量、客户验收 | 完成标准明确，质量合规 |
| 度量 | 有效度量 | 度量指标适宜性 | SMART原则：具体、可测、可达、相关、有时限 |
| 不确定性 | 应对不确定性 | 应对措施适宜性 | 风险四策略：规避、转移、减轻、接受 |

### ⚠️ 绩效域易错点

1. **度量绩效域**的检查方法不是"团队对相互依赖性的理解"，而是"应对措施适宜性"
2. **不确定性绩效域**的储备使用：团队采取步骤主动预防威胁，而非被动等待
3. **团队绩效域**的塔克曼五阶段顺序不能乱：形成→震荡→规范→成熟→解散

## 四、计算题必背公式（5-7分）

### 🔥 挣值分析——案例题必考！

| 指标 | 公式 | 含义 | 判断标准 |
|------|------|------|---------|
| PV | 计划价值 | 应该干多少活 | 基准 |
| EV | 挣值 | 实际干了多少活 | 基准 |
| AC | 实际成本 | 实际花了多少钱 | 基准 |
| **SV** | **EV - PV** | 进度偏差 | >0进度超前，<0进度落后 |
| **CV** | **EV - AC** | 成本偏差 | >0成本节约，<0成本超支 |
| **SPI** | **EV / PV** | 进度绩效指数 | >1进度超前，<1进度落后 |
| **CPI** | **EV / AC** | 成本绩效指数 | >1成本节约，<1成本超支 |

### 🔥 完工预测——记住三个EAC

| 公式 | 适用场景 | 一句话理解 |
|------|---------|-----------|
| EAC = AC + (BAC - EV) | 偏差是偶然的 | 以后能回到正轨 |
| EAC = BAC / CPI | 偏差是常态 | 以后还是这个效率 |
| EAC = AC + (BAC - EV) / (CPI × SPI) | 进度成本都偏差 | 双重修正 |

### 🔥 关键路径法——记住口诀

```
正推取最大（ES = max(紧前EF)）
逆推取最小（LF = min(紧后LS)）
总时差 = LS - ES = LF - EF
自由时差 = min(紧后ES) - EF
关键路径：总时差=0的路径
```

### 🔥 三点估算

```
期望时间 tE = (tO + 4tM + tP) / 6
标准差 σ = (tP - tO) / 6
```

### 🔥 沟通渠道数

```
沟通渠道 = n(n-1) / 2  （n为干系人数量）
```

### 🔥 决策树

```
EMV（期望货币值）= 概率 × 影响
选EMV最大的方案
```

## 八、管理科学基础——必背公式（2-3分）

### 🔥 资金时间价值

```
现值 PV = FV / (1 + r)^n
净现值 NPV = Σ(各年净现金流折现值)
```
- NPV > 0 → 方案可行
- NPV最大的方案最优

### 🔥 投资回收期

```
静态投资回收期 = 累计净现金流量首次为正的年份 - 1 + |上年累计| / 当年净现金流
动态投资回收期 = 累计折现值首次为正的年份 - 1 + |上年累计折现值| / 当年折现值
```

### 🔥 内部收益率IRR

```
IRR > 基准收益率 → 方案可行
```

### 🔥 线性规划

- 三要素：决策变量、目标函数、约束条件
- 求解方法：图解法（两个变量）、单纯形法

### 🔥 决策树

```
EMV = 概率 × 收益（或损失）
选择EMV最大的方案
```

## 十一、案例题救命模板

### 🔥 案例分析万能答题框架

**找问题型题目**：
1. 先看有没有项目管理计划
2. 再看有没有走变更流程
3. 然后看干系人沟通够不够
4. 最后看风险有没有识别

**计算题**：
1. 先找关键路径（总时差=0）
2. 再算挣值（PV/EV/AC）
3. 然后算偏差（SV/CV/SPI/CPI）
4. 最后预测（EAC/ETC）

**找错改错型**：
1. 逐条对照流程看缺了什么
2. 逐条对照ITTO看错了什么
3. 逐条对照原则看违反了什么

## 十二、论文题救命框架

### 🔥 论文万能结构（2000-2500字）

```
1. 项目概述（300字）
   - 项目背景、规模、周期、你的角色

2. 理论过渡（200字）
   - 简述该管理领域的重要性

3. 管理实践（1200字，核心！）
   - 按3-4个过程展开
   - 每个过程：做了什么→用了什么工具→遇到什么问题→怎么解决

4. 总结反思（300字）
   - 成功经验
   - 不足与改进
```
