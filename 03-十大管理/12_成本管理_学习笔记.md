# 12 成本管理

> **核对状态**：✅ 已核对（核对日期：2026-05-16）

## 一、一句话理解
成本管理就是"算好账"——估算项目要花多少钱，制定预算，监控实际花费与预算的偏差，确保项目在批准的预算内完成。

## 二、核心概念与框架

**成本的6种类型（教材原文）**：
- **可变成本**：随生产量、工作量或时间而变（如原材料）
- **固定成本**：不随生产量、工作量或时间变化（如租金）
- **直接成本**：直接归属于项目工作的成本（如项目团队工资、差旅费）
- **间接成本**：多个项目共同分摊的成本（如税金、额外福利）
- **机会成本**：选择某一方案而放弃的其他最佳方案的最大损失
- **沉没成本**：过去决策已发生且无法改变的成本，投资决策时应排除其干扰

| 过程组 | 过程名称 | 核心作用 |
|--------|---------|---------|
| 规划 | 规划成本管理 | 定规矩——成本怎么估算、怎么预算、怎么控制 |
| 规划 | 估算成本 | 算每个活动/工作包需要多少钱 |
| 规划 | 制定预算 | 汇总成本估算，加上储备，形成成本基准 |
| 监控 | 控制成本 | 监控成本偏差，必要时调整 |

## 三、关键知识点逐个讲

### 知识点1：成本基准
- **什么意思**：经过批准的、按时间段分配的项目预算，用来衡量和监控项目成本绩效。
- **为什么重要**：成本基准是"标尺"，用来判断项目是不是超支了。
- **考试怎么考**：选择题考成本基准的组成（成本估算+应急储备）；案例考基准不合理导致的问题。

🏗️ **项目实例**：我的成本基准 = 680万（人力）+ 180万（软件采购）+ 120万（硬件）+ 60万（培训差旅）+ 40万（应急储备）= **1080万元**。注意：应急储备40万包含在成本基准内，项目经理有权使用；但管理储备不包含在基准内，需要高层批准。

### 知识点2：应急储备 vs 管理储备
- **什么意思**：应急储备应对"已知-未知"风险（项目经理有权用）；管理储备应对"未知-未知"风险（需高层批准）。
- **为什么重要**：储备是应对不确定性的缓冲，储备不足风险来了没钱应对。
- **考试怎么考**：高频考点！两者的区别（谁有权用、应对什么风险、是否计入基准）。

🏗️ **项目实例**：
- **应急储备40万**（已知-未知）：我知道对接老旧系统可能出问题，但不确定是哪家、出什么问题——这就是"已知-未知"。后来第3家子公司接口确实出了问题，我直接用应急储备请了外部技术专家驻场支持，不需要请示高层。
- **管理储备**（未知-未知）：项目期间突然出了个新政策要求所有国企系统必须通过等保三级测评——这完全没想到。我向CIO申请动用管理储备来支付测评费用。

**关键公式**：成本基准 = 成本估算 + 应急储备；项目预算 = 成本基准 + 管理储备

### 知识点3：挣值分析（EVM）
- **什么意思**：用三个基本值（PV、EV、AC）和四个指标（SV、CV、SPI、CPI）来衡量项目绩效。
- **为什么重要**：挣值分析是项目管理的"体检表"，能同时反映进度和成本绩效。
- **考试怎么考**：计算题高频考点！给定PV、EV、AC，计算各种指标并分析项目状态。

🏗️ **项目实例**：项目第6个月末，我汇总了数据：
- **PV = 620万**（计划完成数据集成+治理+安全模块开发）
- **EV = 580万**（实际完成了约93%）
- **AC = 610万**（实际花了610万）

计算：
- SV = EV - PV = 580 - 620 = **-40万**（进度滞后40万的工作量）
- CV = EV - AC = 580 - 610 = **-30万**（成本超支30万）
- SPI = 580/620 = **0.94**（进度滞后6%）
- CPI = 580/610 = **0.95**（成本超支5%）

**分析原因**：两家子公司老旧系统接口复杂度超预期，导致集成模块开发滞后、加班成本增加。
**采取措施**：调2名资深开发支援+将进度最慢的2家子公司调为第二批上线。第8个月末CPI回升至0.98，SPI回升至0.97。

### 知识点4：估算方法
- **什么意思**：类比估算（参考历史项目）、参数估算（用公式算）、自下而上估算（先算小再汇总）、三点估算（乐观/最可能/悲观）。
- **为什么重要**：不同方法精度不同，适用场景不同。
- **考试怎么考**：选择题考各种估算方法的特点和适用场景。

🏗️ **项目实例**：
- **类比估算**：我参考去年做的类似数据平台项目（800万），估算本项目约1000万——快但不准，只用于初期粗估
- **参数估算**：按"每个接口开发约5万元"估算，20+个接口≈100万——有公式，比类比准
- **自下而上估算**：把每个WBS工作包的成本逐个估算再汇总——最准但最慢，我用于制定正式预算
- **三点估算**：对接老旧系统，乐观2万/最可能5万/悲观15万，期望=(2+20+15)/6≈6.2万——考虑了不确定性

## 五、ITTO串讲（理解式记忆）

**规划成本管理**
- 它在干什么：定规矩——成本怎么估算、怎么预算、怎么控制。
- 核心输出：成本管理计划。

**估算成本**
- 它在干什么：算每个活动/工作包需要多少钱。
- 工具选择：类比估算（快但不准）、参数估算（用公式）、自下而上估算（准但慢）、三点估算（考虑不确定性）。
- 核心输出：成本估算、估算依据。

**制定预算**
- 它在干什么：汇总成本估算，加上应急储备，形成成本基准；再加上管理储备，形成项目预算。
- **关键公式**：成本基准 = 成本估算 + 应急储备；项目预算 = 成本基准 + 管理储备。
- 核心输出：成本基准、项目资金需求。

**控制成本**
- 它在干什么：监控实际花费与预算的偏差。
- 核心工具：挣值分析（EVM）。
- 核心输出：工作绩效信息、成本预测。

## 六、检查指标（补充理解，非教材原文）

> ⚠️ 以下检查指标为辅助理解内容，非教材原文。教材中成本管理无专门的"检查指标"列表，此部分仅供参考。

| 检查目标 | 检查指标 |
|---------|---------|
| 成本估算准确 | 估算偏差率、估算方法适用度 |
| 预算制定合理 | 应急储备充足率、管理储备占比、资金分配均衡度 |
| 成本控制有效 | CPI（成本绩效指数）、CV（成本偏差）、成本预测准确率 |
| 变更影响可控 | 变更对成本的影响评估完整率、成本变更审批及时率 |

## 七、关键公式（必背！）

| 公式 | 说明 |
|------|------|
| PV（计划价值） | 计划完成工作的预算价值 |
| EV（挣值） | 实际完成工作的预算价值 |
| AC（实际成本） | 实际完成工作的实际成本 |
| SV = EV - PV | 进度偏差，>0超前，<0滞后 |
| CV = EV - AC | 成本偏差，>0节约，<0超支 |
| SPI = EV / PV | 进度绩效指数，>1超前，<1滞后 |
| CPI = EV / AC | 成本绩效指数，>1节约，<1超支 |
| EAC = BAC / CPI | 典型偏差时的完工估算 |
| EAC = AC + (BAC - EV) | 非典型偏差时的完工估算 |
| ETC = EAC - AC | 完工尚需估算 |
| VAC = BAC - EAC | 完工偏差 |
| TCPI = (BAC - EV) / (BAC - AC) | 完工尚需绩效指数，>1难完成，<1易完成 |

**记忆口诀**：
- "EV在前都是好的"——SV、CV、SPI、CPI，EV在减数/分子位置，EV越大越好
- "SPI看进度，CPI看成本"
- ">1都是好事（超前/节约），<1都是坏事（滞后/超支）"

## 八、与其他模块的联动

### 与"进度管理"的联动
进度延误通常导致成本增加（赶工需要加资源），挣值分析同时反映进度和成本绩效。

### 与"范围管理"的联动
范围变更会影响成本，成本超支可能需要缩小范围。

## 九、常见考试出题点

**选择题挖坑**：
- 应急储备vs管理储备的区别（谁有权用、是否计入基准）
- 成本基准vs项目预算的区别（基准+管理储备=预算）
- 各种估算方法的特点（类比快但不准、自下而上准但慢）
- 挣值分析指标的含义（SPI>1超前、CPI<1超支）

**案例分析怎么考**：
- 计算题：给定PV、EV、AC，计算SV、CV、SPI、CPI、EAC、ETC、VAC
- 成本超支分析（原因、影响、应对措施）
- 储备金计算（应急储备、管理储备的估算）

**论文怎么考**：
- "论信息系统项目的成本管理"——按4个过程写
- 重点写：成本估算方法、挣值分析、成本控制措施
